# Техническое задание на внедрение для настольного игрового приложения «404: Kick Not Found»

## Общие сведения
### Полное наименование ИС:  
Настольное игровое приложение «404: Kick Not Found»

Шифр темы: D13-KNF  
Предприятие-разработчик системы: ROFLS Software  
Предприятие-заказчик системы: ProgrNaut Studio  

### Система создается на основании технического задания (ТЗ):  
Техническое задание на разработку игрового приложения
«404: Kick Not Found»

Плановый срок начала работ: 10 сентября 2025 года.  
Плановый срок окончания работ: 20 октября 2025 года.

Автоматизируемая система создается на коммерческой основе.  
Порядок оформления и предъявления заказчику результатов работы по созданию системы определяется после получения начальной версии продукта, в которой должны быть реализованы все основные функции, определенные в ТЗ.

## Назначение и цели создания системы
### Вид автоматизируемой деятельности:
В ИС главным образом автоматизируется проведение игрового процесса, которое сопровождено дополнительными функциями, необходимыми для программы данного вида.

### Перечень автоматизируемых процессов:  
- Проведение главного игрового процесса  
- Отображение меню и разделов приложения, навигация между ними  
- Сохранение и загрузка прогресса игрового процесса  

## Характеристики объекта автоматизации

Программное обеспечение ИС представляет собой настольное игровое приложение, которое предоставляет пользователям автоматизированный игровой процесс в реальном времени.  
Приложение предназначено для пользователей-игроков в качестве развлекательного процесса.

### Краткие сведения о предприятии:  
Предприятие ROFLS Software по заказу ProgrNaut Studio осуществляет совместную разработку, в которой ROFLS Software полностью отвечает за игровой дизайн и художественное обеспечение (изображения, звуковые эффекты и т.д.) продукта. Техническая реализация возложена на оба предприятия.

## Требования к ИС
### Требования к системе в целом:  
- Так как приложение является игровым, общий вид, стиль и оформление должны оставаться едиными и соответствовать продукту.
- Необходимые пользователю данные должны предоставляться в понятном стилизованном графическом представлении.

### Требования к программному обеспечению ИС:  
- Программа должна исправно работать на операционных системах Windows 10, Windows 11 последних стабильных версий.  
- Программа должна работать стабильно и не приводить к экстренным завершениям работы или зависаниям системы.

### Требования к техническому обеспечению АС:  
Аппаратное обеспечение ИС должно по производительности соответствовать минимальным или выше заявленным требованиям:  
Intel Core i5-6200 2,3 ГГц, 4 ГБ ОЗУ, NVIDIA GeForce 940MX 4 ГБ.

## Стадии и этапы разработки

### Разработка должна быть проведена в три стадии:  
- Разработка технического задания.
- Рабочее проектирование.
- Внедрение.

### На стадии рабочего проектирования должны быть выполнены перечисленные этапы работ:  
- Создан дизайн документ игры, в котором подробно описан игровой дизайн продукта.
- Определены оптимальные ограничения для успешной реализации продукта.
- Определены этапы технической разработки.

### Этапы разработки ИС:

#### 1. Разработка дизайн документа игры.  
- Концептуализация. Определение жанра, целевой аудитории и уникального торгового предложения (УТП) игры.
- Создание Геймдизайн-документа (GDD). Детальное описание геймплея, основные и второстепенные механики, баланс. Проектирование игрового мира: лор, сюжет, карта уровней, ключевые локации.
- Создание Арт-концептов. Разработка концепт-артов персонажей, окружения, ключевых объектов и атмосферы игры. Это визуальный эталон для художников.

#### 2. Изучение необходимых для реализации технологий.  
- Прототипирование сложных механик: Создание упрощенных технических прототипов.
- Настройка рабочего окружения. Настройка систем контроля версий Git.

#### 3. Разработка основных механик и систем игры.  
- Реализация базового геймплейного цикла (Core Loop). Программирование основного игрового процесса: движение персонажа, взаимодействие с миром, базовая боевая система.
- Разработка ключевых систем. Программирование: Система сохранений, система загрузки уровней, менеджер игровых состояний, система диалогов, инвентарь.
- Дизайн уровней: Блокинг (серый) уровень на примитивах, который проверяет геймплей и геометрию без финальной графики.

#### 4. Добавление графики и анимаций в приложение.  
- Создание 2D активов. Моделинг, текстурирование персонажей, игровых объектов.
- Создание анимаций. Передвижение, атаки, разрушение объектов.
- Интеграция контента. Размещение финальных моделей на уровнях вместо блокинга. Настройка материалов, освещения и пост-обработки для достижения целевой визуальной стилистики. Интеграция анимаций в систему управления персонажем.
- Работа со звуком. Запись и сведение звуковых эффектов, фоновой музыки.

#### 5. Тестирование и отладка программного продукта.  
- Поиск критических ошибок (вылеты, "зависания", прогресс-блокирующие баги) силами команды разработки.
- Профилирование и устранение "узких мест" для достижения стабильного FPS на целевых платформах.

#### 6. Полишинг проекта (Polishing).  
- Визуальное улучшение. Добавление частиц, эффектов (VFX), динамического освещения, мелких деталей на уровнях. Сглаживание анимаций, добавление реакций на действия игрока.
- Улучшение юзабилити (UX). Настройка плавности интерфейса, добавление анимаций кнопок, улучшение читаемости HUD. Шлифовка системы подсказок и обучения.

### Этапы внедрения ИС:  
- Подготовка к релизу. Создание маркетинговых активов: трейлеры, скриншоты, иконки, описание для магазина. Загрузка сборки и контента в цифровые магазины (Steam, EGS, Itch.io). Подготовка Day-1 патча.
- Релиз и пострелизная поддержка. Мониторинг отзывов игроков в первые часы и дни после выхода. Оперативное исправление критических багов, обнаруженных после релиза.

На этапе подготовки и передачи программы должна быть выполнена работа по подготовке и передаче программы и программной документации в эксплуатацию на объектах заказчика.
